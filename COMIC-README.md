# LLM DOES NOT COMPUTE 🤖

An xkcd-style webcomic about AI, software engineering, and technical absurdity.

**Tagline**: *Artificial Intelligence meets Natural Stupidity*

## Concept

A daily webcomic featuring:
- 🤖 **The Robot**: An LLM-powered stick figure with internal monologue in monospace
- 👤 **The Human**: Regular engineer, often confused
- 🧨 **Simon**: BOFH sysadmin who loves chaos
- 🐱 **The Cat**: Breaks the robot's reasoning

Each day, **two different AI models** generate a comic from the same prompt. Readers **vote** for their favorite variant!

## Features

✅ **Daily Generation**: Cron-triggered at 9 AM UTC
✅ **Dual-Model Comparison**: Llama vs Mistral (or other models)
✅ **Voting System**: Community-driven model evaluation
✅ **Push Notifications**: Browser notifications for new comics
✅ **Archive**: Full history with vote statistics
✅ **Win95 UI**: Retro tabbed interface using 98.css aesthetic
✅ **Cloudflare-Native**: 100% serverless, edge-deployed

## Architecture

```
Frontend (Vue 3)
  ├─ Win95TabContainer.vue (Retro UI)
  ├─ ComicViewer.vue (Daily comic + voting)
  ├─ ComicArchive.vue (Historical archive)
  └─ PushSubscribe.vue (Notification signup)

Backend (Cloudflare)
  ├─ Workers AI (Comic generation)
  ├─ D1 (Metadata, votes, subscriptions)
  ├─ R2 (Image storage)
  ├─ Cron Triggers (Daily automation)
  └─ Web Push (Notifications)

Generation Pipeline
  ├─ comic-generator.ts (Prompt → Script)
  ├─ svg-renderer.ts (Script → SVG)
  └─ _worker.js (Orchestration)
```

## API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/today` | GET | Get today's comic variants + votes |
| `/api/vote` | POST | Vote for variant A or B |
| `/api/archive` | GET | Paginated comic history |
| `/api/subscribe` | POST | Subscribe to push notifications |
| `/api/subscribe` | DELETE | Unsubscribe from push |

## Comic Format

4-panel strip with consistent structure:

**Panel 1**: Setup (Human asks question)
**Panel 2**: Robot internal reasoning (monospace thought bubble)
**Panel 3**: Robot response (speech bubble)
**Panel 4**: Punchline (usually Simon's deadpan comment)

## Example Comic

```
Title: "Prompt Engineering"

Panel 1:
Human: "Robot, explain Kubernetes simply."

Panel 2:
Robot thought bubble:
> request detected
> user comprehension estimate: low
> generating analogy

Panel 3:
Robot: "It's like containers... but with more containers."

Panel 4:
Simon (walking past): "Accurate."
```

## Voting System

Each day presents two variants:
- **Variant A**: Generated by Model A (e.g., Llama 70B)
- **Variant B**: Generated by Model B (e.g., Mistral 7B)

Readers vote for:
- Better humor
- More accurate tech references
- Clearer stick figures
- Overall preference

**Vote deduplication**: IP + User-Agent hash
